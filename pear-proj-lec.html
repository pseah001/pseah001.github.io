2. Aligning on the short & long term direction

Situation

DBS has 3 different loan systems, catering to different loan sizes and their processes. Application of a loan type has to be created in the correct corresponding application. If a small loan is created in medium loan system, application has to be restarted in the correct system. The ask from business was to have a single system to process different loan sizes.​​​​​​​

Task

We had to confirm if having 1 system for different loan types is the correct direction & if its technically feasible​​​​​.

  • Before jumping into solution, we had a discovery session to understand the loan creation flow for each loan type, high level & detailed level

Results

  • Found out that process for small & medium loans is similar. Both go through pre-check to find out if a loan is small or medium. Thus there is a possibility that flow may be combined in future.
  • However, there is no need for a combined flow for big loans. Loan makers can easily identify that a loan application is a big loan at the start and the workflow is very different.

Design

1. From 11 systems to 1 system, 4 steps

Problem

Pain points when proposing a new loan:

  • Manually combine data from 11 different systems & documents
  • Manual input of up to 915 fields
  • Manual calculation of individual information to view as group level
  • Rework cases & repeat process if downstream checks fails
  • Takes around 3h & case may not be converted at the end

Solution

Initally, it may seem there's no sequence with RMs collecting and validating information from multiple systems. But throughout the discovery, we were able to categorise user's main tasks into:

  • Identifying guarantors to form group
  • Checking individual's information to clarify any uncertain points
  • Checking each borrower's existing facilities to calculate the entire group limit

We transformed the current flow into a sequential step so that users can identify necessary information to see all the desired results.

Result

4 steps journey with prevalidation results in 5 mins to determine if a case is worth creating & 15 mins to propose a new loan.

2. Navigation design

Problem

Navigation did not suit workflow as:

  • Jumping of multiple tabs to complete 1 task
  • Many layers to data/clicks to complete 1 task
  • Many under-utilised tabs
  • Multiple external links to extract info or for calculations

Solution

Biggest takeaway

Importance of documentation

It was easy to be information overloaded and feel overwhelmed given the scale of this project. Summaries and documentations were exceptionally important for this project as it provided a consolidated place for us to refresh our memory and focus on the task on hand.
